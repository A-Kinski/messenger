Мессенджер.

Структура:
1. Сервер
2. Клиент для ПК
3. Клиент для телефона (Андроид)

Функции Сервера:
1. Хранение информации о клиентах:
   имя клиента
   адрес клиента
   присутсвие клиента (онлайн/оффлайн)
2. Поддержка связи с другими серверами (при наличии)
3. Хранение таблицы маршрутизации
4. Маршрутизация сообщений
5. Хранение незашифрованных сообщений в БД
6. Создание зашифрованного (vpn) соединения до клиентов
7. Наличие интерфейса для настройки сервера и работы с ним
8. Хранение сообщений, не доставленных до клиента, и их доставка при появленнии клиента в сети

Функции клиента:
1. Формирование и отправка текста
2. Формирование и отправка изображений
3. Формирование и отправка видео
4. Формирование и отправка аудио
5. Возможность шифрованной связи с другим клиентом (по нешифрованному каналу)
6. Возможность создания конференции
7. Отправка на сервер сообщения присутсвия (онлайн/оффлайн)

Техническая составляющая сервера:
1. Разработан на Java и кроссплатформенный
2. БД - MySQL (бесплатная)

Техническая составляющая клиента:
1. Разработан на Java и кроссплатформенный (для ПК)
2. Возможно может хранить БД с сообщениями локально

Что нужно на серверной части для разработки:
1. Установка и настройка Java машины для развертывания приложения
2. Установка и настройка БД MySQL
3. Установка и настройка веб-сервера (пока не знаю зачем, но может пригодится)
4. Возможно установка и настройка DNS, если планируем коннект не только по ip но и по доменному имени
5. Вирутальные машины или Docker-контерйнеры для равертывания нескольких виртуальных клиентов

Что нужно на клиентской части для разработки:
1. Среда программирования
2. Хорошее соединение с сервером при разработке клиента
